This method returns a single extended people record. For official documentation of the
[endpoint](https://thetvdb.github.io/v4-api/#/People/getPeopleExtended), please refer to the Swagger
documentation.

:::caution

There is a known [bug](https://github.com/thetvdb/v4-api/issues/284) in this endpoint that causes it to always return the property `translations` in the response, regardless of whether the `meta` query parameter is set.
:::

## Parameters

| Params      | Type      | Required | Description                                     |
| ----------- | --------- | :------: | ----------------------------------------------- |
| id          | `string`  |   Yes    | The people `id`                                 |
| translation | `boolean` | Optional | Include translation information in the response |

## Examples

### Without translations

```js {4}
import { TheTVDB } from '@untidy/thetvdb';

const client = new TheTVDB('your token');
const data = await client.peopleByIdExtended('312388');
```

### Include translations in the response

```js {4}
import { TheTVDB } from '@untidy/thetvdb';

const client = new TheTVDB('your token');
const data = await client.peopleByIdExtended('312388', true);
```
